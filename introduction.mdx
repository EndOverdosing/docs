## What is Vyla API?

Vyla API is a **Cloudflare Pages API** that scrapes streaming sources from 8 providers concurrently and serves them through a unified JSON interface. Pass any TMDB ID and get back:

<CardGroup cols={2}>
  <Card title="Stream URLs">
    Direct HLS and MP4 sources, sorted by quality — filtered to English audio only
  </Card>
  <Card title="Download Links">
    Pre-built `download_url` and ready-to-run `ffmpeg_command` for every source
  </Card>
  <Card title="Subtitles">
    Deduplicated English subtitle tracks in VTT and SRT, proxied and ready to inject
  </Card>
  <Card title="Embeddable Player">
    One iframe, zero config — fallback logic, subtitles, and HLS.js built in
  </Card>
</CardGroup>

---

## Base URL

```
https://vyla-api.pages.dev
```

All endpoints return `Access-Control-Allow-Origin: *` — no credentials needed.

## Quick Start

<CodeGroup>
```bash Movie Sources
curl https://vyla-api.pages.dev/api/movie?id=27205
```

```bash Enriched + Download-Ready
curl https://vyla-api.pages.dev/api/stream/movie?id=27205
```

```bash Download via ffmpeg
curl https://vyla-api.pages.dev/api/download/movie?id=27205
```

## Endpoint Overview

<AccordionGroup>
  <Accordion title="Scraping Endpoints — raw sources">
    | Endpoint | Description |
    |----------|-------------|
    | `GET /api/movie` | Raw streaming sources for a movie |
    | `GET /api/tv` | Raw streaming sources for a TV episode |
    | `GET /api/scrape` | Single-provider scrape — internal building block |
  </Accordion>

  <Accordion title="Enriched / Download Endpoints — processed sources">
    | Endpoint | Description |
    |----------|-------------|
    | `GET /api/stream/movie` | All providers in parallel · `download_url` · `ffmpeg_command` · deep dedup |
    | `GET /api/stream/tv` | Same, for TV episodes |
    | `GET /api/download/movie` | Alias of `/api/stream/movie` |
    | `GET /api/download/tv` | Alias of `/api/stream/tv` |
  </Accordion>

  <Accordion title="Utility Endpoints">
    | Endpoint | Description |
    |----------|-------------|
    | `GET /api/download` | Proxy-download a single raw video URL |
    | `GET /api/proxy` | Proxy any URL through Cloudflare with CORS + header injection |
    | `GET /api/player` | Embeddable fullscreen HTML player |
  </Accordion>
</AccordionGroup>

## Providers

Sources are scraped concurrently from all 8 providers, filtered to English audio, and sorted by quality before being returned.

| Provider | Type | Notes |
|----------|------|-------|
| 02MovieDownloader | `mp4` / `mkv` | Token-verified direct downloads |
| VixSrc | `hls` | Signed token + expiry, playlist URL format |
| VidSrc | `hls` | Multi-CDN mirror with domain substitution |
| Uembed | `hls` | Multi-API with m3u8 variant resolution |
| VidRock | `hls` / `mp4` | AES-CBC encrypted item IDs |
| RgShows | `mp4` / `hls` | Direct CDN with tripplestream headers |
| VidZee | `hls` | AES-CBC encrypted stream URLs, 14 servers |
| 02Embed | `hls` | Unwrapped from madvid3/02pcembed proxy chain |

## Source Object Reference

<Tabs>
  <Tab title="Raw (/api/movie, /api/tv)">
    ```json
    {
      "url": "https://...",
      "type": "hls | mp4 | mkv",
      "quality": "1080p",
      "provider": "VixSrc",
      "audioTracks": [
        { "language": "eng", "label": "English" }
      ],
      "headers": {
        "Referer": "https://...",
        "Origin": "https://..."
      },
      "vlc_url": "https://..."
    }
    ```

    <Warning>
      `vlc_url` is identical to `url` on raw endpoints — it is **not** proxied. You must forward `headers` yourself when fetching the stream directly.
    </Warning>
  </Tab>

  <Tab title="Enriched (/api/stream/*, /api/download/*)">
    ```json
    {
      "provider": "VixSrc",
      "quality": "1080p",
      "type": "hls",
      "url": "https://vixsrc.to/playlist/...",
      "is_hls": true,
      "referer": "https://vixsrc.to/",
      "ffmpeg_command": "ffmpeg -headers \"...\" -i \"https://...\" -c copy -bsf:a aac_adtstoasc output.mp4",
      "download_url": null,
      "vlc_url": "/api/proxy?url=...&headers=..."
    }
    ```

    <Info>
      `download_url` is `null` for HLS streams — use `ffmpeg_command` instead. `vlc_url` is a pre-built proxied URL ready for VLC or any URL-capable player.
    </Info>
  </Tab>

  <Tab title="Subtitle Object">
    ```json
    {
      "url": "https://...",
      "label": "English",
      "format": "vtt | srt"
    }
    ```

    Subtitle objects appear in the `subtitles` array on every scraping and stream endpoint. See the [Subtitles guide](/api-reference/subtitles) for filtering, deduplication, and usage details.
  </Tab>
</Tabs>
