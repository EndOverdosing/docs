## What is Vyla API?

Vyla API is a **Cloudflare Pages API** that scrapes streaming sources from 6 providers concurrently and serves them through a unified JSON interface. Pass any TMDB ID and get back every working source — already verified and proxied — along with multi-language subtitle tracks.

<CardGroup cols={2}>
  <Card title="Stream URLs">
    All working HLS sources from every provider — verified live before being returned
  </Card>
  <Card title="CORS Proxy">
    Every URL is routed through Cloudflare with full m3u8 segment rewriting — no CORS issues in any browser player
  </Card>
  <Card title="Subtitles">
    Multi-language VTT subtitle tracks included in every movie and TV response, or fetched standalone
  </Card>
  <Card title="Health Check">
    Per-provider health endpoint shows which sources are up and their response times
  </Card>
</CardGroup>

## Base URL

```
https://vyla-api.pages.dev
```

All endpoints return `Access-Control-Allow-Origin: *` — no credentials needed.

## Quick Start

<CodeGroup>
```bash Movie Sources
curl https://vyla-api.pages.dev/api/movie?id=550
```

```bash TV Episode Sources
curl "https://vyla-api.pages.dev/api/tv?id=1396&season=1&episode=1"
```

```bash Movie Subtitles
curl https://vyla-api.pages.dev/api/subtitles/movie/550
```

```bash TV Subtitles
curl https://vyla-api.pages.dev/api/subtitles/tv/76479/1/1
```

```bash Health Check
curl https://vyla-api.pages.dev/api/health
```
</CodeGroup>

## Endpoint Overview

| Endpoint | Description |
|----------|-------------|
| `GET /api/movie` | All working sources + subtitles for a movie |
| `GET /api/tv` | All working sources + subtitles for a TV episode |
| `GET /api/subtitles/movie/<id>` | Subtitle tracks for a movie |
| `GET /api/subtitles/tv/<id>/<season>/<episode>` | Subtitle tracks for a TV episode |
| `GET /api/health` | Per-provider health check |
| `GET /api/test/<id>?source=<key>` | Test a single provider |

## Providers

Sources are fetched from all 6 providers concurrently. Each result is verified with a live stream check — only working sources are returned.

| Key | Label |
|-----|-------|
| `vixsrc` | vixsrc |
| `vidzee` | VidZee |
| `vidnest` | VidNest |
| `vidsrc` | VidSrc |
| `vidrock` | VidRock |
| `videasy` | Videasy |

## Source Object Reference

```json
{
  "source": "vixsrc",
  "label": "vixsrc",
  "url": "/api?url=<encoded_stream_url>&vl=1"
}
```

<ResponseField name="source" type="string">
  Internal provider key.
</ResponseField>

<ResponseField name="label" type="string">
  Human-readable provider name.
</ResponseField>

<ResponseField name="url" type="string">
  Proxied stream URL — routed through `/api` with CORS and m3u8 rewriting applied. Pass directly to HLS.js or any player.
</ResponseField>

---

## Subtitle Object Reference

```json
{
  "label": "English",
  "file": "https://cache.vdrk.site/v1/vtt/movie/550/English.vtt"
}
```

<ResponseField name="label" type="string">
  Language name of the subtitle track.
</ResponseField>

<ResponseField name="file" type="string">
  Direct URL to the VTT subtitle file.
</ResponseField>
