---
title: TV Show
description: Scrape streaming sources for a TV episode by TMDB ID
---

## Endpoint

```
GET /api/tv
```

Scrapes all providers for a TV episode and returns the raw source list. Sources are sorted by quality and filtered to English audio only. No deduplication beyond exact URL matches is applied at this layer.

<Note>
  If you need enriched output with pre-built `download_url`, `ffmpeg_command`, and deeper deduplication, use [`/api/stream/tv`](/api-reference/stream-tv) instead.
</Note>

## Parameters

| Name      | Type   | Required | Default | Description       |
|-----------|--------|----------|---------|-------------------|
| `id`      | string | ✅       | —       | TMDB series ID    |
| `season`  | string | ❌       | `1`     | Season number     |
| `episode` | string | ❌       | `1`     | Episode number    |

## Example Requests

```bash
# Season 1, Episode 1 (defaults)
curl https://vyla-api.pages.dev/api/tv?id=456

# Specific episode
curl https://vyla-api.pages.dev/api/tv?id=456&season=2&episode=3
```

## Source Object Fields

| Field | Type | Description |
|-------|------|-------------|
| `url` | string | Direct stream URL |
| `type` | string | `hls`, `mp4`, or `mkv` |
| `quality` | string | e.g. `1080p`, `720p`, `HD`, `unknown` |
| `provider` | string | Which provider this source came from |
| `audioTracks` | array | Language/label of audio tracks |
| `headers` | object | Required headers to forward when fetching this URL |
| `vlc_url` | string | Same as `url` on this endpoint — not proxied |

## Error Response

```json
{
  "success": false,
  "error": "Missing id"
}
```

<Note>
  TMDB IDs for TV shows refer to the **series**, not individual episodes. Find them at [themoviedb.org](https://www.themoviedb.org).
</Note>

<Warning>
  The `headers` field on each source must be forwarded when fetching the stream directly. Use `/api/stream/tv` to get pre-built `vlc_url` (proxied), `download_url`, and `ffmpeg_command` with headers already encoded.
</Warning>
