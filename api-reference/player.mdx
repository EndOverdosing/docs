## Endpoint

```
GET /api/player
```

Returns a self-contained HTML page with a fullscreen video player. All sources are fetched, proxied, and played automatically with HLS.js. If one source fails or stalls, the player falls back to the next source. Subtitles are injected automatically.

## Parameters

| Name      | Type   | Required | Default  | Description                          |
|-----------|--------|----------|----------|--------------------------------------|
| `type`    | string | ❌       | `movie`  | `movie` or `tv`                      |
| `id`      | string | ✅       | —        | TMDB movie or series ID              |
| `season`  | string | ❌       | `1`      | Season number (TV only)              |
| `episode` | string | ❌       | `1`      | Episode number (TV only)             |
| `nocheck` | string | ❌       | —        | Set to `1` to skip live source checks and use all sources directly |

## Example Requests

```bash
# Movie
https://vyla-api.pages.dev/api/player?type=movie&id=550

# TV episode
https://vyla-api.pages.dev/api/player?type=tv&id=456&season=2&episode=3

# Skip live checks (faster startup, some dead sources may be tried)
https://vyla-api.pages.dev/api/player?type=movie&id=550&nocheck=1
```

## Embed as an iframe

```html
<!-- Movie -->
<iframe
  src="https://vyla-api.pages.dev/api/player?type=movie&id=550"
  width="100%"
  height="500"
  allowfullscreen
  frameborder="0"
></iframe>

<!-- TV Episode -->
<iframe
  src="https://vyla-api.pages.dev/api/player?type=tv&id=456&season=1&episode=1"
  width="100%"
  height="500"
  allowfullscreen
  frameborder="0"
></iframe>
```

## How It Works

1. The player scrapes all providers and performs a live HEAD check on each source (unless `nocheck=1`)
2. Sources that respond successfully are loaded first; dead sources are skipped
3. If all sources fail the check, the full unfiltered list is used as a fallback
4. Sources are loaded in order using HLS.js for HLS streams and the native `<video>` element for MP4
5. A stall timer (8 seconds) auto-advances to the next source if playback doesn't begin
6. All stream and subtitle URLs are routed through `/api/proxy` to handle CORS and required headers
7. Subtitles are attached as `<track>` elements with the first track set as default

## Player Behavior

| Event | Action |
|-------|--------|
| `playing` | Clears the stall timer — source is working |
| `error` | Immediately tries the next source |
| `stalled` | Starts the 8-second stall timer if playback hasn't begun |
| `waiting` | Starts the 8-second stall timer if playback hasn't begun |
| HLS fatal error | Destroys the HLS instance and tries the next source |

## Error Responses

```html
<h2>Missing ?id=</h2>
```

```html
<h2>No sources found.</h2>
```

<Note>
  The player uses HLS.js v1.5.13. On Safari, native HLS is used instead when HLS.js is not supported.
</Note>
