Vyla API is a Cloudflare Pages API that scrapes streaming sources from multiple providers and serves them through a unified JSON interface. Pass any TMDB ID and get back direct stream URLs, subtitles, download links, ffmpeg commands, and an embeddable player.

## Base URL

```
https://vyla-api.pages.dev
```

## Quick Start

Fetch sources for Inception (TMDB ID: 27205):

```bash
curl https://vyla-api.pages.dev/api/movie?id=27205
```

Get download-ready links with ffmpeg commands:

```bash
curl https://vyla-api.pages.dev/api/download/movie?id=27205
```

Embed the player in an iframe:

```html
<iframe
  src="https://vyla-api.pages.dev/api/player?type=movie&id=27205"
  width="100%"
  height="500"
  allowfullscreen
></iframe>
```

## Endpoints

| Endpoint | Description |
|----------|-------------|
| `GET /api/movie` | Streaming sources for a movie |
| `GET /api/tv` | Streaming sources for a TV episode |
| `GET /api/stream/movie` | Alias of `/api/movie` |
| `GET /api/stream/tv` | Alias of `/api/tv` |
| `GET /api/download/movie` | Download-ready sources + ffmpeg commands for a movie |
| `GET /api/download/tv` | Download-ready sources + ffmpeg commands for a TV episode |
| `GET /api/download` | Proxy-download a single video URL directly |
| `GET /api/proxy` | Proxy any URL through Cloudflare with CORS + header injection |
| `GET /api/player` | Embeddable fullscreen HTML player |

## Providers

Sources are scraped concurrently from all providers, deduplicated, filtered to English audio, and sorted by quality before being returned.

| Provider | Type | Notes |
|----------|------|-------|
| 02MovieDownloader | mp4 / mkv | Requires token verification |
| VixSrc | hls | Token + expiry, playlist URL format |
| VidSrc | hls | Multi-CDN mirror deduplication |
| Uembed | hls | Multi-API with m3u8 variant resolution |
| VidRock | hls / mp4 | AES-CBC encrypted item IDs |
| RgShows | mp4 | Direct CDN with tripplestream headers |
| VidZee | hls | AES-CBC encrypted stream URLs |
| 02Embed | hls | Unwrapped from madvid3/02pcembed proxy chain |

## Quality Priority

Sources are ranked and sorted in this order:

```
4K / 2160p > 1440p > 1080p > 720p > 480p > 360p > 240p > HD > Auto > Unknown
```

## CORS

All endpoints return:

```
Access-Control-Allow-Origin: *
Access-Control-Allow-Methods: GET, HEAD, OPTIONS
Access-Control-Allow-Headers: *
```

## Source Object

Every source in a `/api/movie` or `/api/tv` response follows this shape:

```json
{
  "url": "https://...",
  "type": "hls | mp4 | mkv",
  "quality": "1080p",
  "provider": "VixSrc",
  "audioTracks": [
    { "language": "eng", "label": "English" }
  ],
  "headers": {
    "Referer": "https://...",
    "Origin": "https://..."
  }
}
```

The `headers` field must be forwarded when proxying or downloading — omitting them will result in 403 errors from the CDN.

## Download Source Object

Every source in a `/api/download/movie` or `/api/download/tv` response follows this shape:

```json
{
  "provider": "VixSrc",
  "quality": "1080p",
  "type": "hls",
  "url": "https://vixsrc.to/playlist/...",
  "is_hls": true,
  "referer": "https://vixsrc.to/",
  "ffmpeg_command": "ffmpeg -headers \"...\" -i \"https://...\" -c copy -bsf:a aac_adtstoasc output.mp4",
  "download_url": null
}
```

- `download_url` is populated for direct MP4/MKV files — use it as a browser download link
- `download_url` is `null` for HLS streams — use `ffmpeg_command` instead to mux to mp4

## Subtitle Object

```json
{
  "url": "https://...",
  "label": "English",
  "format": "vtt | srt"
}
```

## Notes

- HLS (`.m3u8`) streams and vixsrc `/playlist/` URLs are for playback or ffmpeg only — they cannot be saved as a single file via a browser link
- MP4/MKV sources can be downloaded directly via `/api/download` using the `download_url` field
- All source `headers` must be forwarded to the CDN — pass them as base64 JSON to `/api/proxy` or `/api/download` via the `headers` param
- Duplicate sources (same underlying URL wrapped by different proxy chains) are automatically removed
- CDN mirror duplicates (e.g. same stream path across multiple `tmstr*.` hostnames) are collapsed to one entry
