---
title: "GET /api/stream/movie"
api: "GET https://vyla-api.pages.dev/api/stream/movie"
description: "Enriched movie sources with download URLs, ffmpeg commands, and deduplication across all providers in parallel"
---

Scrapes all 8 providers for a movie **in parallel** — each in an isolated request with its own CPU budget — and returns fully enriched sources. Unlike [`/api/movie`](/api-reference/movie), this endpoint produces significantly more sources, deep-deduplicates them (collapsing provider proxy chains to their real underlying URL), and attaches pre-built `download_url`, `ffmpeg_command`, and proxied `vlc_url` on every source.

`/api/download/movie` is an alias — both endpoints return the identical response.

### Query Parameters

<ParamField query="id" type="string" required>
  TMDB movie ID. Find it at [themoviedb.org](https://www.themoviedb.org), e.g. `themoviedb.org/movie/550` → `550`.
</ParamField>

### Example Request

<CodeGroup>
```bash cURL
curl https://vyla-api.pages.dev/api/stream/movie?id=550
```

```js JavaScript
const res = await fetch("https://vyla-api.pages.dev/api/stream/movie?id=550");
const data = await res.json();

// Download first available MP4
const mp4 = data.sources.find(s => !s.is_hls && s.download_url);
if (mp4) window.location.href = "https://vyla-api.pages.dev" + mp4.download_url;
```

```python Python
import requests

data = requests.get(
  "https://vyla-api.pages.dev/api/stream/movie",
  params={"id": 550}
).json()

print(data["results_found"], "sources found")
for s in data["sources"]:
    print(s["quality"], s["type"], s["provider"])
```

```ts TypeScript
const res = await fetch("https://vyla-api.pages.dev/api/stream/movie?id=550");
const data: StreamMovieResponse = await res.json();

const best = data.sources.find(s => s.quality === "1080p" && !s.is_hls);
```
</CodeGroup>

### Using the Results

<Tabs>
  <Tab title="MP4/MKV — browser download">
    ```js
    const res = await fetch("https://vyla-api.pages.dev/api/stream/movie?id=550");
    const data = await res.json();

    const mp4 = data.sources.find(s => !s.is_hls && s.download_url);
    if (mp4) {
      window.location.href = "https://vyla-api.pages.dev" + mp4.download_url;
    }
    ```
  </Tab>
  <Tab title="HLS — ffmpeg mux">
    Copy the `ffmpeg_command` from any HLS source and run it in your terminal:

    ```bash
    ffmpeg -headers "Referer: https://vixsrc.to/\r\nOrigin: https://vixsrc.to\r\nUser-Agent: Mozilla/5.0...\r\n" \
      -i "https://vixsrc.to/playlist/170060?token=...&expires=...&h=1" \
      -c copy -bsf:a aac_adtstoasc output.mp4
    ```
  </Tab>
  <Tab title="VLC / player URL">
    Use the `vlc_url` field directly — headers are pre-encoded:

    ```bash
    vlc "https://vyla-api.pages.dev/api/proxy?url=...&headers=..."
    ```
  </Tab>
</Tabs>

### Response

<ResponseField name="success" type="boolean" required>
  `true` if scraping succeeded.
</ResponseField>

<ResponseField name="tmdb_id" type="string" required>
  The TMDB ID that was queried.
</ResponseField>

<ResponseField name="results_found" type="number" required>
  Total number of deduplicated sources returned.
</ResponseField>

<ResponseField name="sources" type="EnrichedSource[]" required>
  Fully enriched, deduplicated stream sources sorted by quality.

  <Expandable title="EnrichedSource object">
    <ResponseField name="provider" type="string">
      Which provider this source came from.
    </ResponseField>

    <ResponseField name="quality" type="string">
      e.g. `1080p`, `720p`, `unknown`
    </ResponseField>

    <ResponseField name="type" type="string">
      `hls`, `mp4`, or `mkv`
    </ResponseField>

    <ResponseField name="url" type="string">
      The real, unwrapped stream URL after proxy chain resolution.
    </ResponseField>

    <ResponseField name="is_hls" type="boolean">
      `true` if this is an HLS/m3u8 stream. HLS sources cannot be browser-downloaded directly — use `ffmpeg_command`.
    </ResponseField>

    <ResponseField name="referer" type="string | null">
      The `Referer` header required by the CDN.
    </ResponseField>

    <ResponseField name="ffmpeg_command" type="string">
      Ready-to-run ffmpeg command with all required headers embedded. Works for both HLS and direct sources.
    </ResponseField>

    <ResponseField name="download_url" type="string | null">
      Relative URL for direct browser download — prepend `https://vyla-api.pages.dev`. `null` for HLS sources — use `ffmpeg_command` instead.
    </ResponseField>

    <ResponseField name="vlc_url" type="string">
      Pre-built proxied URL. Open directly in VLC or any URL-capable media player — no additional encoding needed.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="subtitles" type="Subtitle[]" required>
  Deduplicated subtitle array.

  <Expandable title="Subtitle object">
    <ResponseField name="url" type="string">
      Direct URL to the subtitle file.
    </ResponseField>
    <ResponseField name="label" type="string">
      Display name, e.g. `English`, `English [CC]`
    </ResponseField>
    <ResponseField name="format" type="string">
      `vtt` or `srt`
    </ResponseField>
  </Expandable>
</ResponseField>

<Note>
  Duplicate sources (same underlying file wrapped by different provider proxy chains) are automatically removed. CDN mirror duplicates (same stream path across multiple hostnames) are also collapsed to one entry.
</Note>

<Warning>
  HLS/playlist URLs cannot be downloaded as a single file via a browser link. They must be muxed to MP4 using the provided `ffmpeg_command`.
</Warning>
