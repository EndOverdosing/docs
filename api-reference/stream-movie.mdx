title: Stream Movie

## Endpoint

```
GET /api/stream/movie
```

Scrapes all providers for a movie and returns fully enriched sources. Unlike [`/api/movie`](/api-reference/movie), this endpoint applies deep deduplication (collapsing sources wrapped by different proxy chains to their real underlying URL), injects referer headers, generates ready-to-use `download_url` and `ffmpeg_command` fields for every source, and returns a proxied `vlc_url`.

`/api/download/movie` is an alias of this endpoint — they return the same response.

## Parameters

| Name | Type   | Required | Description   |
|------|--------|----------|---------------|
| `id` | string | ✅       | TMDB movie ID |

## Example Request

```bash
curl https://vyla-api.pages.dev/api/stream/movie?id=550
```

## Enriched Source Object Fields

| Field | Type | Description |
|-------|------|-------------|
| `provider` | string | Which provider this source came from |
| `quality` | string | e.g. `1080p`, `720p`, `unknown` |
| `type` | string | `hls`, `mp4`, or `mkv` |
| `url` | string | The real, unwrapped stream URL |
| `is_hls` | boolean | `true` if this is an HLS/m3u8 stream |
| `referer` | string \| null | The `Referer` header required by the CDN |
| `ffmpeg_command` | string | Ready-to-run ffmpeg command with all headers embedded |
| `download_url` | string \| null | Relative URL for direct browser download. `null` for HLS sources |
| `vlc_url` | string | Proxied URL — open in VLC or any player that supports URLs |

## Error Response

```json
{
  "success": false,
  "error": "Missing id"
}
```

## Using the Results

**For MP4/MKV sources** (`is_hls: false`) — use `download_url` as a browser download link:

```js
const res = await fetch("https://vyla-api.pages.dev/api/stream/movie?id=550");
const data = await res.json();

const mp4 = data.sources.find(s => !s.is_hls && s.download_url);
if (mp4) window.location.href = "https://vyla-api.pages.dev" + mp4.download_url;
```

**For HLS sources** (`is_hls: true`) — copy `ffmpeg_command` and run it in your terminal:

```bash
ffmpeg -headers "Referer: https://vixsrc.to/\r\nOrigin: https://vixsrc.to\r\nUser-Agent: Mozilla/5.0...\r\n" \
  -i "https://vixsrc.to/playlist/170060?token=...&expires=...&h=1" \
  -c copy -bsf:a aac_adtstoasc output.mp4
```

<Note>
  Duplicate sources (same underlying file wrapped by different provider proxy chains) are automatically removed. CDN mirror duplicates (same stream path across multiple hostnames) are also collapsed to one entry.
</Note>

<Warning>
  HLS/playlist URLs cannot be downloaded as a single file via a browser link. They must be muxed to mp4 using the provided `ffmpeg_command`.
</Warning>
