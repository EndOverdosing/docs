## Endpoint

```
GET /api/stream/movie
```

`/api/download/movie` is an alias — both return the same response.

Scrapes all 8 providers for a movie **in parallel** — each in an isolated request with its own CPU budget — and returns fully enriched sources. Unlike [`/api/movie`](/api-reference/movie), this endpoint produces significantly more sources, deep-deduplicates them (collapsing provider proxy chains to their real underlying URL), and attaches pre-built `download_url`, `ffmpeg_command`, and proxied `vlc_url` on every source.

---

## Parameters

<ParamField query="id" type="string" required>
  TMDB movie ID.
</ParamField>

---

## Example Request

<CodeGroup>
```bash cURL
curl https://vyla-api.pages.dev/api/stream/movie?id=550
```

```js JavaScript
const res = await fetch("https://vyla-api.pages.dev/api/stream/movie?id=550");
const data = await res.json();

// Download first available MP4
const mp4 = data.sources.find(s => !s.is_hls && s.download_url);
if (mp4) window.location.href = "https://vyla-api.pages.dev" + mp4.download_url;
```

```python Python
import requests

data = requests.get(
  "https://vyla-api.pages.dev/api/stream/movie",
  params={"id": 550}
).json()

print(data["results_found"], "sources found")
```
</CodeGroup>

## Enriched Source Object Fields

<ResponseField name="provider" type="string">
  Which provider this source came from.
</ResponseField>

<ResponseField name="quality" type="string">
  e.g. `1080p`, `720p`, `unknown`
</ResponseField>

<ResponseField name="type" type="string">
  `hls`, `mp4`, or `mkv`
</ResponseField>

<ResponseField name="url" type="string">
  The real, unwrapped stream URL.
</ResponseField>

<ResponseField name="is_hls" type="boolean">
  `true` if this is an HLS/m3u8 stream.
</ResponseField>

<ResponseField name="referer" type="string | null">
  The `Referer` header required by the CDN.
</ResponseField>

<ResponseField name="ffmpeg_command" type="string">
  Ready-to-run ffmpeg command with all headers embedded.
</ResponseField>

<ResponseField name="download_url" type="string | null">
  Relative URL for direct browser download. `null` for HLS sources — use `ffmpeg_command` instead.
</ResponseField>

<ResponseField name="vlc_url" type="string">
  Proxied URL — open in VLC or any player that supports URLs.
</ResponseField>

## Using the Results

<Tabs>
  <Tab title="MP4/MKV — browser download">
    ```js
    const res = await fetch("https://vyla-api.pages.dev/api/stream/movie?id=550");
    const data = await res.json();

    const mp4 = data.sources.find(s => !s.is_hls && s.download_url);
    if (mp4) {
      window.location.href = "https://vyla-api.pages.dev" + mp4.download_url;
    }
    ```
  </Tab>
  <Tab title="HLS — ffmpeg mux">
    Copy the `ffmpeg_command` from any HLS source and run it in your terminal:

    ```bash
    ffmpeg -headers "Referer: https://vixsrc.to/\r\nOrigin: https://vixsrc.to\r\nUser-Agent: Mozilla/5.0...\r\n" \
      -i "https://vixsrc.to/playlist/170060?token=...&expires=...&h=1" \
      -c copy -bsf:a aac_adtstoasc output.mp4
    ```
  </Tab>
  <Tab title="VLC / player URL">
    Use the `vlc_url` field directly — headers are pre-encoded:

    ```
    vlc "https://vyla-api.pages.dev/api/proxy?url=...&headers=..."
    ```
  </Tab>
</Tabs>

<Note>
  Duplicate sources (same underlying file wrapped by different provider proxy chains) are automatically removed. CDN mirror duplicates (same stream path across multiple hostnames) are also collapsed to one entry.
</Note>

<Warning>
  HLS/playlist URLs cannot be downloaded as a single file via a browser link. They must be muxed to MP4 using the provided `ffmpeg_command`.
</Warning>
