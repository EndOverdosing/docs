title: Download
description: Download a video file by URL through the Vyla proxy
---

## Endpoint

```
GET /api/download
```

Fetches a video file and streams it to the browser as a download. Automatically sets `Content-Disposition` so browsers prompt a save dialog. Handles HLS manifest rewriting for `.m3u8` URLs.

## Parameters

| Name       | Type   | Required | Default        | Description |
|------------|--------|----------|----------------|-------------|
| `url`      | string | ✅       | —              | URL-encoded video URL |
| `filename` | string | ❌       | `video.mp4`    | Output filename for the downloaded file |
| `headers`  | string | ❌       | —              | Base64-encoded JSON of extra request headers to forward (e.g. `Referer`, `Origin`) |
| `info`     | string | ❌       | —              | Set to `1` to return a HEAD check as JSON instead of the file |

## Download a File

```bash
curl -L "https://vyla-api.pages.dev/api/download?url=https%3A%2F%2Fexample.com%2Fvideo.mp4&filename=avengers.mp4" \
  -o avengers.mp4
```

Or link to it directly in HTML:

```html
<a href="https://vyla-api.pages.dev/api/download?url=<encoded_url>&filename=movie.mp4">
  Download
</a>
```

## Forwarding Headers

Some CDNs (e.g. `hakunaymatata.com`) require specific `Referer` and `Origin` headers or they return 403. Pass them as a base64-encoded JSON string:

```js
const headers = { "Referer": "https://lok-lok.cc/", "Origin": "https://lok-lok.cc" };
const encoded = btoa(JSON.stringify(headers));
const url = `https://vyla-api.pages.dev/api/download?url=${encodeURIComponent(sourceUrl)}&filename=movie.mp4&headers=${encodeURIComponent(encoded)}`;
```

The `headers` object from each source in the `/api/stream/movie` or `/api/stream/tv` response should always be forwarded this way.

## HEAD Check

Add `&info=1` to perform a HEAD request against the target URL and return its metadata as JSON without streaming the file:

```bash
curl "https://vyla-api.pages.dev/api/download?url=https%3A%2F%2Fexample.com%2Fvideo.mp4&info=1"
```

```json
{
  "success": true,
  "status": 200,
  "url": "https://example.com/video.mp4",
  "content_type": "video/mp4",
  "content_length": "1610612736"
}
```

## Error Responses

```json
{ "success": false, "error": "Missing url param" }
{ "success": false, "error": "Invalid URL" }
{ "success": false, "error": "Fetch failed: <reason>" }
```

<Warning>
  This endpoint is intended for **MP4 and MKV** sources only. HLS (`.m3u8`) streams cannot be saved as a single file — use the `/player` endpoint to stream those instead.
</Warning>

<Note>
  Always forward the `headers` field from source objects. CDN-hosted files from providers like 02MovieDownloader and RgShows will return **Access Denied** without the correct Referer and Origin.
</Note>
