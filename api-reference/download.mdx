---
title: Download
description: Download or inspect a video file by URL
---

## Endpoint

```
GET /api/download
```

Fetches a video file and streams it to the browser as a download. Supports URL unwrapping for third-party proxy URLs and returns proper `Content-Disposition` headers.

## Parameters

| Name       | Type   | Required | Default         | Description                                      |
|------------|--------|----------|-----------------|--------------------------------------------------|
| `url`      | string | ✅       | —               | URL-encoded video URL                            |
| `filename` | string | ❌       | `download.mp4`  | Output filename for the downloaded file          |
| `info`     | string | ❌       | —               | Set to `1` to return JSON metadata instead of the file |

## Download a File

```bash
curl -L "https://vyla-api.pages.dev/api/download?url=https%3A%2F%2Fexample.com%2Fvideo.mp4&filename=inception.mp4" \
  -o inception.mp4
```

Or link to it directly in HTML:

```html
<a href="https://vyla-api.pages.dev/api/download?url=<encoded_url>&filename=movie.mp4">
  Download
</a>
```

## Get File Metadata

Add `&info=1` to return JSON metadata about the file without downloading it:

```bash
curl "https://vyla-api.pages.dev/api/download?url=https%3A%2F%2Fexample.com%2Fvideo.mp4&info=1"
```

### Metadata Response

```json
{
  "success": true,
  "resolved_url": "https://cdn.example.com/video.mp4",
  "original_url": "https://proxy.example.com/video.mp4",
  "filename": "inception.mp4",
  "status": 200,
  "content_type": "video/mp4",
  "content_length": 1610612736,
  "content_length_mb": 1536.0,
  "accept_ranges": "bytes",
  "last_modified": "Mon, 01 Jan 2024 00:00:00 GMT",
  "download_url": "/api/download?url=...&filename=inception.mp4"
}
```

## Error Responses

```json
{
  "success": false,
  "error": "Missing url parameter",
  "usage": {
    "download": "/api/download?url=<encoded_url>&filename=<name.mp4>",
    "info": "/api/download?url=<encoded_url>&info=1"
  }
}
```

```json
{ "success": false, "error": "Fetch failed: <reason>" }
{ "success": false, "error": "Upstream returned 403" }
```

<Warning>
  This endpoint is intended for **MP4 and MKV** sources only. HLS (`.m3u8`) streams cannot be downloaded as a single file — use the `/api/player` endpoint to stream those instead.
</Warning>

<Note>
  URLs wrapped in third-party proxies are automatically unwrapped before fetching, so you can pass raw source URLs directly from the `/api/movie` or `/api/tv` response.
</Note>
