title: Download
description: Download a video file by URL, or get enriched download-ready sources by TMDB ID

Vyla API has two separate download systems:

- `/api/download` — proxy-download a single raw video URL directly
- `/api/stream/movie` and `/api/stream/tv` (also aliased as `/api/download/movie` and `/api/download/tv`) — scrape all sources for a title and return enriched sources with `download_url` and `ffmpeg_command`

---

## Download by URL

```
GET /api/download
```

Fetches a video file and streams it to the browser as a download. Automatically sets `Content-Disposition` so browsers prompt a save dialog. For HLS URLs, returns an ffmpeg command instead.

### Parameters

| Name       | Type   | Required | Default        | Description |
|------------|--------|----------|----------------|-------------|
| `url`      | string | ✅       | —              | URL-encoded video URL |
| `filename` | string | ❌       | `video.mp4`    | Output filename for the downloaded file |
| `headers`  | string | ❌       | —              | Base64-encoded JSON of extra request headers (`Referer`, `Origin`) |
| `info`     | string | ❌       | —              | Set to `1` to return a HEAD check as JSON instead of downloading |
| `ffmpeg`   | string | ❌       | —              | Set to `1` to always return an ffmpeg command instead of downloading |

### Download a File

```bash
curl -L "https://vyla-api.pages.dev/api/download?url=https%3A%2F%2Fexample.com%2Fvideo.mp4&filename=movie.mp4" \
  -o movie.mp4
````

Or as a browser download link:

```html
<a href="https://vyla-api.pages.dev/api/download?url=<encoded_url>&filename=movie.mp4&headers=<base64_headers>">
  Download
</a>
```

### Forwarding Headers

CDNs like `hakunaymatata.com` and `tripplestream.online` require specific `Referer` and `Origin` headers or they return 403. Pass them as base64-encoded JSON:

```js
const headers = { "Referer": "https://lok-lok.cc/", "Origin": "https://lok-lok.cc" };
const encoded = encodeURIComponent(btoa(JSON.stringify(headers)));
const url = `https://vyla-api.pages.dev/api/download?url=${encodeURIComponent(sourceUrl)}&filename=movie.mp4&headers=${encoded}`;
```

The `download_url` field from `/api/stream/movie` and `/api/stream/tv` already has this encoded and ready to use.

### HLS / Playlist URLs

If the URL contains `.m3u8` or `/playlist/`, the endpoint automatically returns an ffmpeg command instead of attempting a stream download:

```json
{
  "success": true,
  "ffmpeg_command": "ffmpeg -headers \"Referer: https://vixsrc.to/\\r\\nOrigin: https://vixsrc.to\\r\\nUser-Agent: Mozilla/5.0...\\r\\n\" -i \"https://vixsrc.to/playlist/170060?token=...\" -c copy -bsf:a aac_adtstoasc output.mp4",
  "url": "https://vixsrc.to/playlist/170060?token=...",
  "referer": "https://vixsrc.to/"
}
```

### HEAD Check

Add `&info=1` to check a URL's metadata without downloading:

```bash
curl "https://vyla-api.pages.dev/api/download?url=https%3A%2F%2Fexample.com%2Fvideo.mp4&info=1"
```

```json
{
  "success": true,
  "status": 200,
  "url": "https://example.com/video.mp4",
  "content_type": "video/mp4",
  "content_length": "1610612736",
  "is_hls": false
}
```

### Error Responses

```json
{ "success": false, "error": "Missing url param" }
{ "success": false, "error": "Invalid URL" }
{ "success": false, "error": "All fetch attempts failed" }
{ "success": false, "error": "Upstream returned 403" }
```

## Download by TMDB ID — Movie

```
GET /api/download/movie
GET /api/stream/movie
```

These are aliases of each other. Both scrape all providers for a movie and return every source enriched with a ready-to-use `download_url` or `ffmpeg_command`. See the [Stream Movie](/api-reference/stream-movie) reference for the full response shape and field descriptions.

### Parameters

| Name | Type   | Required | Description   |
| ---- | ------ | -------- | ------------- |
| `id` | string | ✅        | TMDB movie ID |

### Example Request

```bash
curl https://vyla-api.pages.dev/api/download/movie?id=550
```

---

## Download by TMDB ID — TV

```
GET /api/download/tv
GET /api/stream/tv
```

These are aliases of each other. Same as the movie endpoint but for TV episodes. See the [Stream TV](/api-reference/stream-tv) reference for the full response shape and field descriptions.

### Parameters

| Name      | Type   | Required | Default | Description    |
| --------- | ------ | -------- | ------- | -------------- |
| `id`      | string | ✅        | —       | TMDB series ID |
| `season`  | string | ❌        | `1`     | Season number  |
| `episode` | string | ❌        | `1`     | Episode number |

### Example Request

```bash
curl "https://vyla-api.pages.dev/api/download/tv?id=456&season=1&episode=1"
```

## Using the Results

**For MP4/MKV sources** (`is_hls: false`) — use `download_url` directly as a browser `<a href>` download link. It already has all headers encoded.

**For HLS sources** (`is_hls: true`) — copy the `ffmpeg_command` and run it in your terminal:

```bash
ffmpeg -headers "Referer: https://vixsrc.to/\r\nOrigin: https://vixsrc.to\r\nUser-Agent: Mozilla/5.0...\r\n" \
  -i "https://vixsrc.to/playlist/170060?token=...&expires=...&h=1" \
  -c copy -bsf:a aac_adtstoasc output.mp4
```

<Warning>
  HLS/playlist URLs cannot be downloaded as a single file via a browser link. They must be muxed to mp4 using the provided `ffmpeg_command`.
</Warning>

<Note>
  Duplicate sources (same underlying file wrapped by different provider proxy chains) are automatically removed. CDN mirror duplicates (same stream path across multiple hostnames) are also collapsed.
</Note>
