## Finding TMDB IDs

Go to [themoviedb.org](https://www.themoviedb.org) and search for any title. The ID is in the URL:

| URL | TMDB ID | Title |
|-----|---------|-------|
| `themoviedb.org/movie/550` | `550` | Fight Club |
| `themoviedb.org/movie/27205` | `27205` | Inception |
| `themoviedb.org/tv/456` | `456` | The Simpsons |
| `themoviedb.org/tv/1396` | `1396` | Breaking Bad |

## Build Your Own Player

<Steps>
  <Step title="Fetch all working sources and subtitles">
    <CodeGroup>
    ```bash Movie
    curl https://vyla-api.pages.dev/api/movie?id=550
    ```

    ```bash TV Episode
    curl "https://vyla-api.pages.dev/api/tv?id=1396&season=1&episode=1"
    ```
    </CodeGroup>

    Every source in the response has already been verified live — no dead URLs. Subtitles are included automatically.
  </Step>

  <Step title="Play with HLS.js and load subtitles">
    ```html
    <video id="player" controls></video>
    <script src="https://cdn.jsdelivr.net/npm/hls.js@1.5.13/dist/hls.min.js"></script>
    <script>
    async function loadMovie(tmdbId) {
      const res = await fetch(`https://vyla-api.pages.dev/api/movie?id=${tmdbId}`);
      const data = await res.json();

      if (!data.sources.length) return;

      const source = data.sources[0];
      const video = document.getElementById("player");

      const hls = new Hls();
      hls.loadSource(source.url);
      hls.attachMedia(video);

      hls.on(Hls.Events.MANIFEST_PARSED, () => {
        data.subtitles.forEach(sub => {
          const track = document.createElement("track");
          track.kind = "subtitles";
          track.label = sub.label;
          track.src = sub.file;
          video.appendChild(track);
        });
        video.play();
      });
    }

    loadMovie(550);
    </script>
    ```

    <Note>
      The `url` field on every source is already proxied and CORS-safe — pass it directly to HLS.js with no additional encoding. Subtitle `file` URLs are direct VTT links and can be used as `<track>` elements.
    </Note>
  </Step>

  <Step title="Try the next source on failure">
    ```js
    async function loadWithFallback(tmdbId) {
      const res = await fetch(`https://vyla-api.pages.dev/api/movie?id=${tmdbId}`);
      const data = await res.json();
      const sources = data.sources;
      let index = 0;

      function trySource() {
        if (index >= sources.length) return;
        const video = document.getElementById("player");
        const hls = new Hls();

        hls.on(Hls.Events.ERROR, (_, err) => {
          if (err.fatal) { hls.destroy(); index++; trySource(); }
        });

        hls.loadSource(sources[index].url);
        hls.attachMedia(video);
        hls.on(Hls.Events.MANIFEST_PARSED, () => video.play());
      }

      trySource();
    }
    ```
  </Step>
</Steps>

## Fetch Subtitles Standalone

You can fetch subtitle tracks independently without fetching stream sources:

```bash
# Movie subtitles
curl https://vyla-api.pages.dev/api/subtitles/movie/550

# TV episode subtitles
curl https://vyla-api.pages.dev/api/subtitles/tv/1396/1/1
```

```json
[
  { "label": "English", "file": "https://cache.vdrk.site/v1/vtt/movie/550/English.vtt" },
  { "label": "Spanish", "file": "https://cache.vdrk.site/v1/vtt/movie/550/Spanish.vtt" },
  { "label": "French",  "file": "https://cache.vdrk.site/v1/vtt/movie/550/French.vtt" }
]
```

## Check Provider Health

```bash
curl https://vyla-api.pages.dev/api/health
```

```json
{
  "status": "ok",
  "timestamp": "2024-01-01T00:00:00.000Z",
  "tmdb": true,
  "cache": 4,
  "probe_id": "550",
  "sources": {
    "vixsrc":  { "ok": true,  "ms": 812 },
    "vidzee":  { "ok": true,  "ms": 1204 },
    "vidnest": { "ok": true,  "ms": 980 },
    "vidsrc":  { "ok": false, "ms": null },
    "vidrock": { "ok": true,  "ms": 1540 },
    "videasy": { "ok": true,  "ms": 1100 }
  }
}
```

`status` is `"ok"` only when every provider passes. `"degraded"` means at least one is down — the rest still work.

## Test a Single Provider

Useful for debugging or checking why a specific source isn't appearing:

```bash
# Test vixsrc for Fight Club
curl "https://vyla-api.pages.dev/api/test/550?source=vixsrc"

# Test vidzee for Breaking Bad S1E1
curl "https://vyla-api.pages.dev/api/test/1396?season=1&episode=1&source=vidzee"
```

```json
{
  "source": "vixsrc",
  "id": "550",
  "s": null,
  "e": null,
  "ok": true,
  "url": "/api?url=<encoded>&vl=1",
  "raw_url": "https://vixsrc.to/playlist/...",
  "elapsed_ms": 923,
  "error": null
}
```
