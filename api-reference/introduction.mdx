Vyla API is a Cloudflare Pages API that scrapes streaming sources from multiple providers and serves them through a unified JSON interface. Pass any TMDB ID and get back direct stream URLs, subtitles, and an embeddable player.

## Base URL

```
https://vyla-api.pages.dev
```

## Quick Start

Fetch sources for Inception (TMDB ID: 27205):

```bash
curl https://vyla-api.pages.dev/api/stream/movie?id=27205
```

Embed the player in an iframe:

```html
<iframe
  src="https://vyla-api.pages.dev/player?type=movie&id=27205"
  width="100%"
  height="500"
  allowfullscreen
></iframe>
```

## Providers

Sources are scraped concurrently from all providers, deduplicated, filtered to English audio, and sorted by quality before being returned.

| Provider | Type | Notes |
|----------|------|-------|
| 02MovieDownloader | mp4 / hls | Requires token verification |
| VixSrc | hls | Token + expiry from page HTML |
| VidSrc | hls | Multi-domain template resolution |
| Uembed | hls | Multi-API with m3u8 variant resolution |
| VidRock | hls / mp4 | AES-CBC encrypted item IDs |
| RgShows | mp4 | Decodes 02pcembed proxy to extract direct URL and headers |
| VidZee | hls | AES-CBC encrypted stream URLs |
| 02Embed | hls | Rewritten via madvid3.xyz HLS proxy |

## Quality Priority

Sources are ranked and sorted in this order:

```
4K / 2160p > 1440p > 1080p > 720p > 480p > 360p > 240p > HD > Auto > Unknown
```

## CORS

All endpoints return:

```
Access-Control-Allow-Origin: *
Access-Control-Allow-Methods: GET, HEAD, OPTIONS
Access-Control-Allow-Headers: *
```

## Source Object

Every source in a response follows this shape:

```json
{
  "url": "https://...",
  "type": "hls | mp4 | mkv",
  "quality": "1080p",
  "provider": "VixSrc",
  "audioTracks": [
    { "language": "eng", "label": "English" }
  ],
  "headers": {
    "Referer": "https://...",
    "Origin": "https://..."
  }
}
```

The `headers` field is present on most sources and must be forwarded when proxying or downloading — omitting them will result in 403 Access Denied from the CDN.

## Subtitle Object

```json
{
  "url": "https://...",
  "label": "English",
  "format": "vtt | srt"
}
```

## Notes

- HLS (`.m3u8`) streams are for playback only, not direct download
- MP4/MKV sources can be downloaded via `/api/download` when available
- All source `headers` must be forwarded to the CDN — pass them as base64 JSON to `/api/proxy` or `/api/download` via the `headers` param
- If one provider fails, sources from the remaining providers are still returned
```
