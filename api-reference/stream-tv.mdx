## Endpoint

```
GET /api/stream/tv
```

Scrapes all providers for a TV episode and returns fully enriched sources. Unlike [`/api/tv`](/api-reference/tv), this endpoint fans out to all providers in parallel â€” each in an isolated request â€” so every provider gets its own CPU budget. Sources are deeply deduplicated, with pre-built `download_url`, `ffmpeg_command`, and proxied `vlc_url` fields on every source.

`/api/download/tv` is an alias of this endpoint â€” they return the same response.

## Enriched Source Object Fields

| Field | Type | Description |
|-------|------|-------------|
| `provider` | string | Which provider this source came from |
| `quality` | string | e.g. `1080p`, `720p`, `unknown` |
| `type` | string | `hls`, `mp4`, or `mkv` |
| `url` | string | The real, unwrapped stream URL |
| `is_hls` | boolean | `true` if this is an HLS/m3u8 stream |
| `referer` | string \| null | The `Referer` header required by the CDN |
| `ffmpeg_command` | string | Ready-to-run ffmpeg command with all headers embedded |
| `download_url` | string \| null | Relative URL for direct browser download. `null` for HLS sources |
| `vlc_url` | string | Proxied URL â€” open in VLC or any player that supports URLs |

## Error Response

```json
{
  "success": false,
  "error": "Missing id"
}
```

<Note>
  Duplicate sources (same underlying file wrapped by different provider proxy chains) are automatically removed.
</Note>

<Warning>
  HLS/playlist URLs cannot be downloaded as a single file via a browser link. They must be muxed to mp4 using the provided `ffmpeg_command`.
</Warning>
