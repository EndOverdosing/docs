Every `/api/movie`, `/api/tv`, `/api/stream/movie`, and `/api/stream/tv` response includes a `subtitles` array alongside `sources`.

## Subtitle Object

```json
{
  "url": "https://vdrk.b-cdn.net/cache/movie/550/English.vtt",
  "label": "English",
  "format": "vtt"
}
```

<ResponseField name="url" type="string" required>
  Direct URL to the subtitle file. May require a `Referer` header — route through `/api/proxy` to handle automatically.
</ResponseField>

<ResponseField name="label" type="string" required>
  Display name — e.g. `English`, `English [CC]`, `French`, `Bengali`
</ResponseField>

<ResponseField name="format" type="string" required>
  `vtt` or `srt`. Most subtitles are VTT; SRT files come primarily from the hakunaymatata CDN.
</ResponseField>

## Filtering & Deduplication

<Tabs>
  <Tab title="Filtering">
    Only **English subtitles** are returned. A subtitle is kept if any of the following match:

    - `label` contains `"english"` or `"en"`
    - `language` code matches `eng`, `en`, `en-us`, or `en-gb`
    - label is `"unknown"`
  </Tab>
  <Tab title="Deduplication">
    The same subtitle file is often returned by multiple providers through different proxy chains. Vyla deduplicates by the **underlying URL**, so you never get the same track listed twice.

    For example, all of these collapse to one entry:

    ```
    https://vdrk.b-cdn.net/cache/movie/550/English.vtt
    https://madvid3.xyz/api/hls-proxy?url=...vdrk.b-cdn.net/cache/movie/550/English.vtt...
    ```

    VixSrc subtitle tokens (`token`, `expires`, `edge`) are stripped before deduplication so the same rendition from different scrape calls doesn't produce duplicates.
  </Tab>
</Tabs>

## Common Labels

| Label | Description |
|-------|-------------|
| `English` | Standard English subtitles |
| `English [CC]` | Closed captions — includes sound descriptions |
| `English - English` | Duplicate label from VidRock CDN naming |
| `English 1`, `English 2` ... | Multiple English tracks (e.g. SDH vs standard) |
| `French` | French subtitles |
| `French - Canadian French` | Canadian French variant |
| `Bengali` | Bengali subtitles from VidRock |

---

## VTT vs SRT

| Format | Notes |
|--------|-------|
| `vtt` | Native browser support — works directly in `<track src>` |
| `srt` | Requires conversion for browser playback. Use [subtitle.js](https://github.com/gsantiago/subtitle.js) or a proxy that converts on the fly |

## Using Subtitles

<Tabs>
  <Tab title="Native video element">
    Subtitle URLs from VixSrc and some CDN sources require a `Referer` header. Route them through `/api/proxy`:

    ```js
    const res = await fetch("https://vyla-api.pages.dev/api/movie?id=550");
    const data = await res.json();

    const video = document.getElementById("player");

    data.subtitles.forEach((sub, i) => {
      const track = document.createElement("track");
      track.kind = "subtitles";
      track.label = sub.label;
      track.srclang = sub.label.toLowerCase().startsWith("fr") ? "fr" : "en";
      track.src = "https://vyla-api.pages.dev/api/proxy?url=" + encodeURIComponent(sub.url);
      if (i === 0) track.default = true;
      video.appendChild(track);
    });
    ```
  </Tab>
  <Tab title="HLS.js">
    HLS.js doesn't use `<track>` elements — inject subtitles after the manifest is parsed:

    ```js
    hls.on(Hls.Events.MANIFEST_PARSED, () => {
      data.subtitles.forEach((sub, i) => {
        const track = document.createElement("track");
        track.kind = "subtitles";
        track.label = sub.label;
        track.srclang = "en";
        track.src = "https://vyla-api.pages.dev/api/proxy?url=" + encodeURIComponent(sub.url);
        if (i === 0) track.default = true;
        video.appendChild(track);
      });

      video.play();
    });
    ```
  </Tab>
</Tabs>

<Note>
  The `/api/player` endpoint handles all of this automatically — subtitles are proxied and injected as `<track>` elements with no configuration needed.
</Note>
