---
title: "GET /api/movie"
api: "GET https://vyla-api.pages.dev/api/movie"
description: "Scrape streaming sources for a movie by TMDB ID"
---

Scrapes all providers for a movie and returns the raw source list. Sources are sorted by quality and filtered to English audio only. No deduplication beyond exact URL matches is applied at this layer.

<Note>
  For enriched output with pre-built `download_url`, `ffmpeg_command`, and deeper deduplication across all providers in parallel, use [`GET /api/stream/movie`](/api-reference/stream-movie) instead.
</Note>

### Query Parameters

<ParamField query="id" type="string" required>
  TMDB movie ID. Find it at [themoviedb.org](https://www.themoviedb.org) — it's the number in the URL, e.g. `themoviedb.org/movie/550` → `550`.
</ParamField>

### Example Request

<CodeGroup>
```bash cURL
curl https://vyla-api.pages.dev/api/movie?id=550
```

```js JavaScript
const res = await fetch("https://vyla-api.pages.dev/api/movie?id=550");
const data = await res.json();

console.log(data.sources); // array of stream sources
```

```python Python
import requests

data = requests.get(
  "https://vyla-api.pages.dev/api/movie",
  params={"id": 550}
).json()
print(data["sources"])
```

```ts TypeScript
const res = await fetch("https://vyla-api.pages.dev/api/movie?id=550");
const data: MovieResponse = await res.json();
```
</CodeGroup>

### Response

<ResponseField name="success" type="boolean" required>
  `true` if scraping succeeded, `false` otherwise.
</ResponseField>

<ResponseField name="results_found" type="number" required>
  Total number of sources returned.
</ResponseField>

<ResponseField name="sources" type="Source[]" required>
  Array of stream source objects, sorted by quality descending.

  <Expandable title="Source object">
    <ResponseField name="url" type="string">
      Direct stream URL. Signed and may expire.
    </ResponseField>

    <ResponseField name="type" type="string">
      `hls`, `mp4`, or `mkv`
    </ResponseField>

    <ResponseField name="quality" type="string">
      e.g. `1080p`, `720p`, `HD`, `unknown`
    </ResponseField>

    <ResponseField name="provider" type="string">
      Which provider this source came from.
    </ResponseField>

    <ResponseField name="audioTracks" type="AudioTrack[]">
      Available audio tracks.

      <Expandable title="AudioTrack object">
        <ResponseField name="language" type="string">
          ISO language code, e.g. `eng`
        </ResponseField>
        <ResponseField name="label" type="string">
          Human-readable label, e.g. `English`
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="headers" type="object">
      Required headers to forward when fetching this URL directly. **Must be included** or the CDN will return 403.

      <Expandable title="Headers object">
        <ResponseField name="Referer" type="string">
          Required `Referer` header value.
        </ResponseField>
        <ResponseField name="Origin" type="string">
          Required `Origin` header value.
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="vlc_url" type="string">
      Same as `url` on this endpoint — not proxied. Use `/api/stream/movie` to get a pre-built proxied `vlc_url`.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="subtitles" type="Subtitle[]" required>
  Array of subtitle objects. See [Subtitles](/api-reference/subtitles) for full details.

  <Expandable title="Subtitle object">
    <ResponseField name="url" type="string">
      Direct URL to the subtitle file.
    </ResponseField>
    <ResponseField name="label" type="string">
      Display name, e.g. `English`, `English [CC]`
    </ResponseField>
    <ResponseField name="format" type="string">
      `vtt` or `srt`
    </ResponseField>
  </Expandable>
</ResponseField>

<Warning>
  The `headers` field on each source **must be forwarded** when fetching the stream directly. Pass them as base64 JSON to `/api/proxy` via the `headers` param, or use `/api/stream/movie` which handles header encoding in `vlc_url` automatically.
</Warning>
