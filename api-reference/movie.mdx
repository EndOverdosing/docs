title: Movie
description: Scrape streaming sources for a movie by TMDB ID
---

## Endpoint

```
GET /api/movie
```

Scrapes all providers for a movie and returns the raw source list. Sources are sorted by quality and filtered to English audio only. No deduplication beyond exact URL matches is applied at this layer.

<Note>
  If you need enriched output with pre-built `download_url`, `ffmpeg_command`, and deeper deduplication, use [`/api/stream/movie`](/api-reference/stream-movie) instead.
</Note>

## Parameters

| Name | Type   | Required | Description   |
|------|--------|----------|---------------|
| `id` | string | ✅       | TMDB movie ID |

## Example Request

```bash
curl https://vyla-api.pages.dev/api/movie?id=550
```

## Source Object Fields

| Field | Type | Description |
|-------|------|-------------|
| `url` | string | Direct stream URL |
| `type` | string | `hls`, `mp4`, or `mkv` |
| `quality` | string | e.g. `1080p`, `720p`, `HD`, `unknown` |
| `provider` | string | Which provider this source came from |
| `audioTracks` | array | Language/label of audio tracks |
| `headers` | object | Required headers to forward when fetching this URL |
| `vlc_url` | string | Same as `url` on this endpoint — not proxied |

## Error Response

```json
{
  "success": false,
  "error": "Missing id"
}
```

<Warning>
  The `headers` field on each source must be forwarded when fetching the stream directly. Pass them as base64 JSON to `/api/proxy` via the `headers` param, or use `/api/stream/movie` which handles header encoding in `vlc_url` automatically.
</Warning>

<Note>
  TMDB movie IDs can be found at [themoviedb.org](https://www.themoviedb.org). The ID is in the URL — e.g. `themoviedb.org/movie/550` → ID is `550`.
</Note>
