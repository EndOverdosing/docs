---
title: "GET /api/proxy"
api: "GET https://vyla-api.pages.dev/api/proxy"
description: "Proxy any video URL through Cloudflare with CORS, header injection, and full m3u8 rewriting"
---

Proxies any URL through Cloudflare Workers. Handles CORS, injects required `Referer` and `Origin` headers, and **rewrites m3u8 playlists** so all segment URLs are also proxied automatically — the entire HLS stream plays without CORS issues in any browser player.

Also accepts `HEAD` requests for live source checking.

### Query Parameters

<ParamField query="url" type="string" required>
  URL-encoded target URL to proxy.
</ParamField>

<ParamField query="headers" type="string">
  Base64-encoded JSON object of additional request headers. Used to forward `Referer`, `Origin`, or any other header required by the CDN.

  ```js
  const headers = { Referer: "https://vixsrc.to/", Origin: "https://vixsrc.to" };
  const encoded = btoa(JSON.stringify(headers));
  // &headers=eyJSZWZlcmVyIjoiaHR0cHM6...
  ```
</ParamField>

### Example Request

<CodeGroup>
```bash Plain video URL
curl "https://vyla-api.pages.dev/api/proxy?url=https%3A%2F%2Fexample.com%2Fvideo.mp4"
```

```bash With custom headers
curl "https://vyla-api.pages.dev/api/proxy?url=https%3A%2F%2Fexample.com%2Fstream.m3u8&headers=eyJSZWZlcmVyIjoiaHR0cHM6Ly9leGFtcGxlLmNvbSJ9"
```

```js JavaScript
const streamUrl = "https://vixsrc.to/playlist/170060?token=...";
const headers = { Referer: "https://vixsrc.to/", Origin: "https://vixsrc.to" };

const proxiedUrl = "https://vyla-api.pages.dev/api/proxy"
  + "?url=" + encodeURIComponent(streamUrl)
  + "&headers=" + encodeURIComponent(btoa(JSON.stringify(headers)));

// Use directly with HLS.js
const hls = new Hls();
hls.loadSource(proxiedUrl);
hls.attachMedia(video);
```

```python Python
import base64, json, urllib.parse

stream_url = "https://vixsrc.to/playlist/170060?token=..."
headers = {"Referer": "https://vixsrc.to/", "Origin": "https://vixsrc.to"}

encoded_headers = base64.b64encode(json.dumps(headers).encode()).decode()
proxy_url = (
  "https://vyla-api.pages.dev/api/proxy"
  + "?url=" + urllib.parse.quote(stream_url)
  + "&headers=" + urllib.parse.quote(encoded_headers)
)
```
</CodeGroup>

### M3U8 Rewriting

When the proxied URL returns an m3u8 playlist, **all segment and variant URLs inside it are automatically rewritten** to also go through `/api/proxy`. This ensures the entire HLS stream plays without CORS issues.

```
Original segment:  https://cdn.example.com/seg001.ts
Rewritten segment: https://vyla-api.pages.dev/api/proxy?url=https%3A%2F%2Fcdn.example.com%2Fseg001.ts
```

If the original request included a `headers` param, it is preserved and forwarded for every rewritten segment URL automatically.

### Automatic Header Injection

For known providers, the proxy injects the correct headers even without the `headers` param:

| Domain | Auto-injected headers |
|--------|-----------------------|
| `tripplestream.online`, `hlmv-files` | `Referer: https://www.rgshows.ru`, `Origin: https://www.rgshows.ru` |
| Any other URL | `Referer` and `Origin` set to the URL's own origin |

### Encoding Headers

```js
const headers = { Referer: "https://vixsrc.to/", Origin: "https://vixsrc.to" };
const encoded = btoa(JSON.stringify(headers));
// eyJSZWZlcmVyIjoiaHR0cHM6Ly92aXhzcmMudG8vIiwiT3JpZ2luIjoiaHR0cHM6Ly92aXhzcmMudG8ifQ==
// Pass as: &headers=eyJSZWZlcmVy...
```

### Security — SSRF Protection

Private IP ranges and localhost are blocked:

<AccordionGroup>
  <Accordion title="Blocked hosts">
    - `127.0.0.1`, `::1`, `0.0.0.0`
    - `10.x.x.x`, `192.168.x.x`, `172.16–31.x.x`
    - `.internal`, `.local` domains
    - `169.254.169.254` (GCP metadata endpoint)
  </Accordion>
</AccordionGroup>

### Response

On success, the proxy streams the upstream response body directly with appropriate CORS headers. No JSON envelope.

On error, a JSON object is returned:

<ResponseField name="error" type="string">
  Description of the failure.
</ResponseField>

<Tabs>
  <Tab title="Success">
    The upstream response body is streamed directly. Content-Type, Content-Length, and other relevant headers are forwarded. CORS headers are added.
  </Tab>
  <Tab title="Error responses">
    ```json
    { "error": "Missing url parameter" }
    ```
    ```json
    { "error": "Invalid url" }
    ```
    ```json
    { "error": "Blocked host" }
    ```
    ```json
    { "error": "Fetch failed: <reason>" }
    ```
    ```json
    { "error": "Upstream error 403" }
    ```
  </Tab>
</Tabs>

<Note>
  The proxy is used internally by `/api/player` — all stream and subtitle URLs served to the player are already routed through `/api/proxy`.
</Note>

<Note>
  The `vlc_url` field returned by `/api/stream/movie` and `/api/stream/tv` is a **pre-built proxy URL** — pass it directly to VLC or any URL-capable player without additional encoding.
</Note>
